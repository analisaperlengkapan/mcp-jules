# Security Policy

## Supported versions

Security fixes are applied to the latest release line only.

## Reporting a vulnerability

Do not disclose security issues publicly.

If this repository is hosted on GitHub, open a private security advisory.
Otherwise, contact the project maintainer through the private channel used for the project.

Include as much detail as possible:

- affected version
- steps to reproduce
- impact assessment
- any proof of concept or logs

## Response process

The maintainers will review reports, confirm impact, and coordinate a fix before public disclosure.
